terug

Styling die je toepast wordt toegepast in een bepaalde volgorde. Het kan voorkomen dat je styling toepast en vervolgens niet het gewenste resultaat ziet. Cascade en Inheritance zijn belangrijke concepten om te begrijpen hoe styling werkt. Lees hier in de officiële documentatie meer over cascade en inheritance. Zoek ook andere referenties om het concept te begrijpen. Leg in eigen woorden uit wat met cascade en inheritance wordt bedoeld. Maak voorbeelden om je antwoord uit te leggen.

Cascade

dit betekent dat de volgorde van CSS-regels ertoe doet; wanneer twee regels van toepassing zijn die dezelfde specificiteit hebben, wordt degene die als laatste in de CSS komt, degene die zal worden gebruikt.

voorbeeld

h6 { color: red; } h6 { color: blue; }
h6
h6

Specificity

Specificiteit is hoe de browser beslist welke regel van toepassing is als meerdere regels verschillende selectors hebben, maar nog steeds van toepassing kunnen zijn op hetzelfde element. Het is in feite een maatstaf voor hoe specifiek de selectie van een selector zal zijn: Een elementenselector is minder specifiek - het selecteert alle elementen van dat type die op een pagina verschijnen - en krijgt dus een lagere score. Een klassenkiezer is specifieker - hij selecteert alleen de elementen op een pagina die een specifieke klasse-attribuutwaarde hebben - en krijgt dus een hogere score.

h1 class="main-heading">This is my heading./h1 .main-heading { color: red; } h1 { color: blue; }

Inheritance

Overerving moet ook in deze context worden begrepen - sommige CSS-eigenschapswaarden die zijn ingesteld op bovenliggende elementen, worden geërfd door hun onderliggende elementen, en andere niet.

div { color: blueviolet; }
span binnen de div
